Construction Site Safety Assessment System 工地現場安全自動判釋系統架構提案
總覽 營造法規知識圖譜 模組 資料格式

以法規核心串接表單欄位與歷史資料的工地安全判釋架構

系統以法規/內規為核心，將違規類型、改善措施、表單欄位與歷史查驗紀錄串成可追溯圖譜，再由頂層 Construction Regulation Knowledge Graph（CRKG，營造法規知識圖譜）串接，完成 Safety Alert（安全警示）、改善建議與自動填表。

照片輸入 違規範圍標記 Visual Language Model（VLM，視覺語言模型）觀察 多類別法規檢索 Safety Alert（安全警示）閾值判定 查核表自動填寫

上到下流程圖

1. 照片與工程資料 現場照片、工項、日期、施工階段、承包商。
2. 圖譜檢索、法規比對與注意錨點 從法規/內規出發，沿表單連到歷史資料，完成適用規則比對、表單對應與 VLM 判釋錨點建立。
法規/內規
表單
歷史資料
國家法規 地方規定 工地內規
3. 脈絡引導式 Visual Language Model（VLM，視覺語言模型）判讀 視覺語言模型依注意錨點做全圖判讀，並由 VLM grounding 產生疑似違規區域標記。
4. 人工查核 由職安人員檢視 VLM 標註、照片證據與表單填寫內容，確認是否採納或修正。
5. 輸出結果 標記後照片、Safety Alert Report（安全警示報告）、改善建議、已填查核表。

第 2 步同時完成法規/內規、表單與歷史資料的圖譜檢索，建立 VLM 判釋錨點；VLM 負責看圖與標記，輸出端再產生標註照片與表單內容。

完整 Graph 示意圖

整體主幹是一個三層法規脈絡圖：第一層是法規/內規，第二層是表單，第三層是歷史資料；這整個圖譜從上方連到 Visual Language Model（VLM，視覺語言模型），作為判斷錨點。主流程則是新拍照片進入 VLM，再輸出標註照片或新表單內容。

照片標記輸出設計

每個違規判定都應能回指到照片上的證據範圍。照片標記由 Visual Language Model grounding（視覺語言模型定位）依注意錨點產生。

BOX-001 疑似未戴安全帽
BOX-002 高處作業
BOX-003 疑似未戴安全帽
POLY-001 施工架平台

標記結果如何進入判釋

  • 法規節點連到「安全帽」查核欄位，該欄位的歷史資料提示過往查核重點，VLM 因此產生 BOX-001、BOX-003 作為疑似個人防護具證據。
  • 工項脈絡提示「施工架作業」，VLM 標示 BOX-002 供高處作業規則判定。
  • 法規節點對應的表單欄位提示需查核平台/護欄，VLM 標示 POLY-001 檢查踏板與臨邊風險。

標記圖層

  • bbox（bounding box，矩形框）：人員、安全帽、反光背心、機具。
  • polygon（多邊形區域）：開口、臨邊、施工架平台、危險區域。
  • label：疑似違規類型、標註範圍與人工查核狀態。
  • link：標記範圍連到 Observation（OBS，結構化觀察）與 Violation Decision（違規判定）。

標記產生方式

  • 法規/內規 → 表單欄位 → 歷史查驗紀錄檢索：產生應注意項目與提示詞。
  • Visual Language Model grounding（視覺語言模型定位）：依脈絡錨點產生 bounding box / polygon 與標籤。

資料格式範例

建議系統內部用 JSON 傳遞，使 Visual Language Model（VLM，視覺語言模型）觀察、法規判定與表單填寫可測試、可追蹤、可人工複核。

1. Attention Anchor

由法規/內規、表單與歷史資料產生，告訴 VLM 這張照片要優先看什麼。

ANCHOR-001
2. Photo Annotation

VLM 依錨點在照片上標出 bbox / polygon，作為可回看的影像證據。

anchor_id → ANCHOR-001
3. Observation（OBS）

把多個標記整理成一筆視覺觀察；OBS 只描述看見什麼，不直接判定違規。

linked_annotations → BOX-001, BOX-003
4. Violation Decision

把 OBS、照片標記與法規/表單依據串起來，形成疑似違規、改善與填表內容。

source_observations → OBS-001
